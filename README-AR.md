# دليل النظام باللغة العربية

## نظرة عامة
هذا النظام يقدم إدارة عملاء عقارية عربية بالكامل مع تخطيط RTL وأدوار واضحة (مالك، قائد فريق، مبيعات) ومراحل متابعة ثابتة، مع تنبيهات وتقارير.

## تخصيص الألوان
تم تعريف الألوان في ملف:
- web/app/globals.css

يمكنك تعديل المتغيرات التالية:
- --primary-600
- --success-500
- --warning-400
- --danger-500
- --stage-1 إلى --stage-4

## إضافة أو حذف مراحل العمل
المراحل الافتراضية تظهر عبر مكوّن LeadProgress. لتعديلها:
- في الواجهة: مرّر مصفوفة المراحل المطلوبة إلى LeadProgress
- في التحكم بالمراحل: عدّل stageLabels و stageKeys داخل StageControls

### استخدام LeadProgress
- المكوّن يقبل props: stages و activeIndex و onStageChange
- يرسل onStageChange عند الضغط على مرحلة معينة

## تخصيص الأدوار والصلاحيات
- الصلاحيات تتحكم بها طبقة الـ RBAC في الباك اند.
- يمكن إضافة دور جديد عبر إنشاء Role وربطه بالمستخدمين داخل النظام.

## تخصيص التنبيهات
- تنبيهات النظام تعتمد على وحدات notifications و reminder.
- تنبيهات WhatsApp تعتمد على وحدة whatsapp وإعدادات القوالب.

## تعطيل الحركات والأنيميشن
النظام يحترم إعداد النظام `prefers-reduced-motion` تلقائيًا.
يمكنك تفعيل هذا الخيار من إعدادات النظام لديك، وسيتم تعطيل تحريك الخلفيات وتقدم المراحل.

## تنبيهات وإشعارات
- تنبيهات داخل النظام عبر وحدة الإشعارات.
- دعم WhatsApp متاح عبر وحدة whatsapp في الباك اند.

## تخصيص الروابط والبيئة
- Frontend: http://localhost:3000
- Backend: http://localhost:4000
- proxy للـ API عبر /api

## مصفوفة دعم المتصفحات
- Chrome (آخر إصدارين)
- Edge (آخر إصدارين)
- Firefox (آخر إصدارين)
- Safari 15+

## أدوار النظام
### المالك
- إدارة المستخدمين وترقياتهم
- توزيع العملاء على قادة الفرق
- متابعة جميع المراحل والتنبيهات والتقارير

### قائد الفريق
- توزيع العملاء على المبيعات
- متابعة الأداء والمراحل
- استقبال تنبيهات إكمال المراحل

### المبيعات
- تنفيذ مراحل العميل الأربعة خلال أسبوع
- إرسال التنبيهات بعد كل مرحلة
- طلب تمديد محدود عند التأخر
